# Security Policy

## Reporting a vulnerability

If you discover a security issue in Kin2 Workforce, please report it responsibly.

**Contact:** kin@kin2serviceslimited.com  
**Subject line:** `[SECURITY] <brief description>`

Please include:
- A short description of the issue
- Affected area, page, or API endpoint
- Steps to reproduce (if safe to share)
- Screenshots or supporting context if available
- Your assessment of severity and impact

Do not open a public GitHub issue for security vulnerabilities.

## Response commitments

| Stage | Target |
|---|---|
| Acknowledgement | Within 48 hours of receipt |
| Initial assessment | Within 5 business days |
| Resolution or mitigation | Dependent on severity — critical issues prioritised |
| Disclosure coordination | We will work with you on timing before any public disclosure |

## Scope

**In scope:**
- Authentication and session management
- Multi-tenant data isolation
- API endpoints and key management
- GDPR / personal data handling
- Payroll and HMRC submission flows
- GPS clock-in and location data

**Out of scope:**
- Findings on third-party services (Stripe, Twilio, SendGrid, HMRC)
- Social engineering attacks
- Physical security
- Denial of service
